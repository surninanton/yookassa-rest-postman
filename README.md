# YooKassa API — Postman Collection

Неофициальная Postman-коллекция для работы с **API ЮKassa (YooKassa) v3**. Собрана по тому же принципу, что и [bitrix24-rest-postman](https://github.com/surninanton/bitrix24-rest-postman): все методы разложены по папкам, авторизация и идемпотентность настроены централизованно, у каждого запроса — реальный пример тела из документации и авто-тесты.

- Документация API: <https://yookassa.ru/developers/api>
- Быстрый старт: <https://yookassa.ru/developers/payment-acceptance/getting-started/quick-start>

## Что внутри

| Файл | Назначение |
|---|---|
| `YooKassa-API.postman_collection.json` | Готовая коллекция для импорта в Postman |
| `build_collection.py` | Генератор коллекции (правьте описание методов здесь) |
| `LICENSE` | MIT |

**29 запросов** в 11 папках:

- **Платежи** — создать, список, информация, подтвердить (capture), отменить (cancel)
- **Возвраты** — создать, список, информация
- **Чеки** — создать, список, информация
- **Счета (Invoices)** — создать, информация
- **Способы оплаты** — создать (сохранить), информация
- **Выплаты** — создать, информация
- **Сделки (SafeDeal)** — создать, список, информация
- **Персональные данные** — создать, информация
- **Самозанятые** — зарегистрировать, информация
- **Участники СБП** — список банков
- **Магазин и вебхуки** — информация о магазине (`/me`), создать/список/удалить вебхук

## Настройка

1. Импортируйте `YooKassa-API.postman_collection.json` в Postman (**Import → File**).
2. Откройте коллекцию → вкладка **Variables** и заполните:

| Переменная | Значение |
|---|---|
| `base_url` | `https://api.yookassa.ru/v3` (уже задано) |
| `shop_id` | Идентификатор магазина из личного кабинета ЮKassa |
| `secret_key` | Секретный ключ из личного кабинета ЮKassa |

3. Готово. Авторизация и заголовки подставляются автоматически (см. ниже).

Где взять `shop_id` и `secret_key`: личный кабинет ЮKassa → **Настройки → API-ключи**. Для тестов используйте ключ тестового магазина.

## Как устроена авторизация

- **Basic Auth** настроена на уровне коллекции: логин — `{{shop_id}}`, пароль — `{{secret_key}}`. Все запросы наследуют её автоматически.
- **`Idempotence-Key`** для всех `POST`-запросов задан значением `{{$guid}}` — Postman генерирует новый UUID при каждой отправке. Это защищает от дублей при повторной отправке (см. [идемпотентность](https://yookassa.ru/developers/using-api/interaction-format#idempotence)).
- **ID подставляются между запросами.** После «Создать платёж» ответный `id` сохраняется в переменную `payment_id`, и запросы «Информация о платеже», «Подтвердить», «Отменить», «Создать возврат» и т.д. используют его автоматически. Аналогично для `refund_id`, `receipt_id`, `invoice_id`, `payout_id`, `deal_id` и других.

## Типичный сценарий

1. **Платежи → Создать платёж** — в ответе возьмите `confirmation.confirmation_url` и откройте его в браузере, чтобы оплатить (тестовая карта: `5555 5555 5555 4444`, любой будущий срок и CVC).
2. **Платежи → Информация о платеже** — дождитесь статуса `succeeded` (или `waiting_for_capture` при `capture: false`).
3. При двухстадийной оплате — **Подтвердить платёж (capture)** или **Отменить платёж (cancel)**.
4. При необходимости — **Возвраты → Создать возврат**.

Статусы вместо ручного опроса лучше получать через **вебхуки** (папка «Магазин и вебхуки»).

## Тесты

У каждого запроса есть авто-тесты: проверка статуса `200` и того, что ответ — JSON. Запросы на создание дополнительно сохраняют `id` объекта в переменную коллекции.

## Пересборка коллекции

Описание методов и примеры тел находятся в `build_collection.py`. После правок:

```bash
python3 build_collection.py
```

Скрипт перезапишет `YooKassa-API.postman_collection.json`.
